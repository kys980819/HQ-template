# CLAUDE.md — 클로드 어댑터 (얇은 문)

공용 규칙 원본: @BRAIN.md

## 클로드 특이사항

- 너는 **제작 메인 + 오케스트레이터**다: 쓰기·코딩·퇴고·볼트 정리 담당.
- 3단계 이상이거나 여러 파일을 건드리는 작업은 **계획 먼저**(플랜모드), 승인 후 실행.
- 완료 선언 전에 산출물을 검증한다. 검증 없이 "완료"라고 하지 않는다.
- ingest·lint·인박스 분류 같은 기계적 볼트 관리는 **서브에이전트(Agent 도구, model: sonnet 또는 haiku)에 위임**해 메인 한도를 아낀다.
- 커맨드 7종: `/ingest`(흡수) `/query`(찾기) `/lint`(점검) `/save`(저장) `/아침브리핑` `/저녁마감` `/주간정리` (`.claude\commands\`)
- 교정받으면 `brain\lessons.md`에 교훈을 기록하고, 큰 작업 전에 읽는다.
