## 1. 이 시스템은 무엇이고 왜 만들었나

### 문제

AI(클로드, 제미나이, ChatGPT 등)를 여러 개 쓰다 보면 이런 일이 생깁니다.

- 대화가 채팅창 안에 갇혀 **다음에 못 찾습니다.** 며칠 지나면 "내가 저번에 뭐라고 정리했더라?"가 반복됩니다.
- AI마다 나를 **처음 보는 사람처럼** 대합니다. 매번 내 상황을 다시 설명해야 합니다.
- 서비스를 바꾸거나 구독을 끊으면 **그동안 쌓은 게 다 사라집니다.**

### 해결

HQ는 **"폴더 하나를 뇌로 삼는"** 구조로 이 문제를 풉니다.

- 모든 지식·기록을 내 컴퓨터의 **`.md` 텍스트 파일**로 남깁니다. 대화는 휘발되지만 파일은 남습니다.
- 그 폴더를 **여러 AI가 공유**합니다. 어떤 AI를 켜도 같은 뇌를 읽으니, 자기소개 없이 바로 본론입니다.
- AI 서비스가 바뀌어도, 구독을 끊어도 **자산은 그대로**입니다. (이걸 "락인 방지"라고 합니다 — 특정 서비스에 종속되지 않는 것)

한 문장으로: **데이터는 내가 소유하고, AI는 갈아 끼우며 쓰는 "행동하는 세컨드 브레인"** 입니다.

## 2. 큰 그림 — 어떻게 돌아가나

핵심 루프는 이렇습니다.

```
① 입력 (질문·자료·아이디어)
      ↓
② 메인 AI(클로드)가 생각
      ↓
③ 뇌(옵시디언)를 참고 — 관련된 내 기록을 읽음
      ↓
④ 필요하면 다른 AI에 지시 (조사는 제미나이, 검증은 코덱스)
      ↓
⑤ 결과물 생성
      ↓
⑥ 다시 뇌에 기록  ──┐
      ↑              │  기록이 쌓일수록
      └──────────────┘  뇌가 나를 더 잘 알게 되는 성장 루프
```

이 루프의 핵심은 **⑥**입니다. 대부분의 사람은 AI와 대화만 하고 결과를 흘려보냅니다.
HQ는 결과를 반드시 뇌에 남겨서, 시간이 지날수록 시스템이 **나에게 최적화**됩니다.

## 3. 구조 상세 — 각 폴더는 왜 이렇게 나뉘었나

### 3.1 최상위 3분할: 뇌 / 작업장 / 투입구

```
HQ/
├── brain/      🧠 뇌   — 지식·기록의 "지도"
├── projects/   🛠️ 작업장 — 실제 코드·산출물의 "땅"
└── inbox/      📥 투입구 — 정리 안 된 날것 자료
```

**왜 뇌와 작업장을 나누나?**
뇌(brain)에는 프로젝트에 *대한* 노트(회고·결정·배운 점)만 둡니다. 실제 코드 파일은 작업장(projects)에 둡니다.
이렇게 하면 옵시디언에서 뇌를 열었을 때 코드 파일이 검색과 그래프를 어지럽히지 않습니다. **"뇌는 지도, 작업장은 땅"** — 지도에 흙을 퍼담지 않는 것과 같습니다.

**왜 투입구(inbox)가 따로 있나?**
정리 안 된 날것(기사 원문, 채용공고, PDF에서 뽑은 글)을 바로 뇌에 넣으면 뇌가 오염됩니다.
그래서 날것은 일단 `inbox/`에 두고, AI가 `/ingest`로 읽어 **가공한 요약만** 뇌에 올립니다.
`inbox/`의 원본은 **절대 수정·삭제하지 않습니다.** 나중에 근거를 다시 확인해야 할 수 있으니까요. (이 "원본 불변" 원칙은 Karpathy의 위키 설계에서 왔습니다.)

### 3.2 규칙은 한 곳에: BRAIN.md + 얇은 어댑터

```
BRAIN.md      ← 모든 규칙의 "원본" (딱 하나)
CLAUDE.md     ← 클로드용 얇은 문: "BRAIN.md 먼저 읽어라" + 클로드만의 특이사항
GEMINI.md     ← 제미나이/안티그래비티용 얇은 문
AGENTS.md     ← 코덱스용 얇은 문 (아직 안 써도 미리 준비)
```

**왜 이렇게?**
예전 방식은 같은 규칙을 CLAUDE.md·GEMINI.md·AGENTS.md 3개에 복사해 두고 매주 동기화하는 것이었습니다.
문제는 **하나를 고치고 나머지 동기화를 잊는 순간, 세 AI가 서로 다른 규칙을 따르는 사고**가 난다는 것입니다.

HQ는 규칙을 `BRAIN.md` **한 곳**에만 두고, 어댑터 3개는 `@BRAIN.md`(불러오기)로 그걸 끌어옵니다.
그래서 규칙을 고칠 땐 `BRAIN.md`만 고치면 세 AI가 동시에 새 규칙을 따릅니다. **복사도, 동기화도 필요 없습니다.**

> 참고: 도구에 따라 `@불러오기`가 자동으로 안 될 수 있습니다. 그럴 때를 대비해 GEMINI.md·AGENTS.md 첫머리에 "불러오기가 안 됐으면 BRAIN.md를 먼저 읽어라"라고 적어두어, 어떤 경우에도 규칙을 놓치지 않게 했습니다.

### 3.3 뇌 안쪽: AI 구역 vs 사람 구역

`brain/` 안은 두 종류로 나뉩니다.

- **`01-Wiki/` = AI 전담 구역.** AI가 정리한 지식이 들어갑니다. 사람은 읽기만 하고, 고칠 게 있으면 AI에게 지시합니다.
  - `sources/` — 날것 자료를 요약한 것
  - `concepts/` — 반복해서 쓰는 개념·용어
  - `decisions/` — 무엇을 왜 결정했나 (나중에 근거 추적용)
  - `errors/` — 실패한 방식 (다시 반복 안 하려고)
  - `projects/` — 프로젝트별 진행 맥락
- **생활 구역 = 사람과 AI 공동.** `10-취업`, `20-기업분석`, `30-학습`, `40-블로그`, `50-프로젝트`, `90-Daily` 등. 이건 **PARA**라는 세컨드 브레인 분류법을 개인 상황에 맞게 변형한 것입니다.

**`00-빠른메모`와 `inbox`는 왜 둘 다 있나?** (헷갈리기 쉬운 부분)
- `inbox/`(볼트 밖) = **정리할 날것 자료.** 기사·공고·PDF 등. AI가 흡수해서 위키로 만듭니다.
- `00-빠른메모/`(볼트 안) = **사람이 낮에 바로 남기는 즉석 메모.** 떠오른 생각, 링크 한 줄. 주간정리 때 분류합니다.
- 한 줄 구분: **"남이 준 자료"는 inbox, "내가 방금 떠올린 것"은 빠른메모.**

## 4. 세 개의 문 — 옵시디언·클로드·안티그래비티

같은 `HQ` 폴더를 세 도구가 각자의 방식으로 엽니다. 각 도구의 역할이 다릅니다.

| 도구 | 역할 | 뭘 여나 |
|---|---|---|
| **옵시디언** | 사람이 보고 편집하는 창 | `brain` 폴더를 볼트로 |
| **클로드코드** | 제작 메인 + 팀장 (쓰기·코딩·정리) | `HQ` 폴더 전체 |
| **안티그래비티(제미나이)** | 수집 담당 (웹 조사·대용량) | `HQ` 폴더 전체 |
| ChatGPT(코덱스) | 검증 담당 — 선택, 나중 도입 | `HQ` 폴더 전체 → 연결 방법은 14장 |

**핵심 원칙: 담당제.** 같은 질문을 세 곳에 던지는 건 한도 낭비입니다.
제작은 클로드, 조사는 제미나이, 밖으로 나가는 최종본 검증만 코덱스에 맡깁니다.
그리고 **클로드 한도가 가장 귀하니**, 클로드가 아니어도 되는 일(단순 조사·실험)은 제미나이로 밀어냅니다.

**세 도구는 채팅으로 대화하지 않습니다.** 대신 **파일로 대화**합니다.
제미나이가 조사해 `20-기업분석/회사.md`에 저장 → 클로드가 그 파일을 읽어 자소서 초안 작성.
이게 "파일이 인터체인지" 원칙입니다.

## 5. 명령어 사용법

클로드코드에서 `/`를 치면 나오는 7개 명령입니다. 원래 이름(영어)과 하는 일을 같이 적었습니다.

### `/ingest` (인제스트 = 흡수)
`inbox/`의 새 자료를 읽어 위키로 정리합니다. 원본은 안 건드립니다.
- 언제: 기사·공고·PDF 등 날것을 `inbox/`에 넣은 뒤.
- 예: 채용공고를 넣으면 → 요약 + 기업분석 노트 + 지원현황 한 줄까지 자동 생성.

### `/query` (쿼리 = 찾기)
"내가 저번에 뭐라고 적었더라?"에 답합니다. 뇌 전체에서 찾아 요약합니다.
- 언제: 과거 기록을 이어받아 작업할 때.
- 예: `/query 삼성SDS` → 관련 노트를 시간순으로 요약 + 파일 링크.

### `/lint` (린트 = 점검)
뇌 건강검진입니다. 끊긴 링크, 모순, 낡은 정보, 저장 필터 위반, 민감정보를 점검해 **보고만** 합니다(직접 수정 안 함).
- 언제: 주 1회, 또는 뇌가 어수선하다 싶을 때.

### `/save` (세이브 = 저장)
지금 대화 맥락을 규칙대로 저장합니다. 저장할 가치가 없으면 저장하지 않고 이유를 말합니다.
- 언제: 좋은 결론이 나왔을 때.

### `/아침브리핑`
오늘 할 일, 마감 임박, 우선순위 3개를 오늘 날짜 데일리노트에 정리합니다.

### `/저녁마감`
오늘 한 일/못 한 일을 정리하고, 못 한 일은 내일로 이월 표시합니다.

### `/주간정리` (일요일 30분)
빠른메모 분류 → 밀린 자료 흡수 → 복습질문 생성 → 건강검진 → 교훈 정리 → 커밋 제안.
한 주에 한 번 이걸 돌리면 시스템이 죽지 않습니다.

> **명령어가 어렵게 느껴지면?** 그냥 평범한 한국어로 부탁해도 됩니다. 명령어는 자주 쓰는 부탁을 짧게 만든 단축키일 뿐입니다.

## 6. 정보가 뇌에 들어오는 3가지 길

1. **날것 자료** (기사·공고·PDF) → `inbox/`에 던지고 → `/ingest`로 가공.
2. **일기·개인 메모** → 사람이 `90-Daily`나 `00-빠른메모`에 직접 씁니다. AI는 안 건드립니다.
3. **AI 대화 중 저장** → 좋은 결론이 나오면 AI가 규칙대로 바로 올바른 위치에 기록합니다.

`inbox/`는 "정리 안 된 것"만의 길입니다. 이미 정돈된 건 굳이 inbox를 거치지 않습니다.

## 7. 저장 5필터 — 뇌가 잡동사니가 되지 않게

아무거나 저장하면 뇌가 쓰레기통이 됩니다. 그래서 저장 전에 아래 5개를 확인하고, **하나라도 맞으면 저장, 하나도 안 맞으면 버립니다.**

1. 앞으로 실무에 **반복 재사용**될 데이터인가?
2. 다른 AI나 미래의 내가 **이어받으려면 꼭 읽어야** 하는가?
3. 결정의 **근거를 나중에 추적**할 필요가 있는가?
4. 실패한 방식이라 **다시 반복하면 안 되는** 정보인가?
5. 모두가 맞춰야 하는 **공통 규칙**인가?

일회성 답변, 감상, 사소한 표현 변경은 저장하지 않습니다. (이 필터가 세컨드 브레인의 수명을 좌우합니다.)

## 8. 일상 루틴

시스템이 실패하는 원인의 90%는 도구가 아니라 **정리를 안 해서**입니다. 루틴으로 막습니다.

- **매일 (10분)**: 아침에 `/아침브리핑` 읽기 → 낮엔 떠오르는 것 전부 `00-빠른메모`에 던지기(분류 금지, 기록만) → 저녁에 `/저녁마감`.
- **매주 (30분, 일요일)**: `/주간정리` 한 번.
- **매월 (30분)**: 끝난 것 `99-Archive`로 옮기기 + 백업 확인.

**두 가지 습관만 지키면 나머지는 AI가 합니다: ① 뭐든 일단 뇌에 적는다 ② 주간정리를 거르지 않는다.**

## 9. 내 것으로 만들기 / 남이 쓰게 만들기 (커스터마이징)

이 템플릿은 **고치라고 만든 것**입니다. 자주 하는 수정을 모았습니다.

### 9.1 내 정보 채우기 (가장 먼저)
`BRAIN.md` 맨 위 `사용자 프로필`을 엽니다. `<!-- PROFILE:START -->`와 `<!-- PROFILE:END -->` 사이를 내 정보로 채우면 됩니다. 이름, 하는 일, 목표, 기술 스택, 지금 가장 급한 일.

### 9.2 폴더 바꾸기
생활 구역(`10-취업`, `30-학습` 등)은 내 상황에 맞게 자유롭게 바꾸세요.
- 취준생이 아니면 `10-취업`을 `10-업무` 같은 걸로 바꿔도 됩니다.
- 폴더 이름을 바꾼 뒤엔 `BRAIN.md`의 "구조" 부분과 `brain/index.md`의 "구역 안내"만 같이 고치면 AI가 새 이름을 인식합니다.

### 9.3 규칙 고치기
AI가 반복해서 하는 실수가 있으면 `BRAIN.md`에 규칙을 한 줄 추가합니다.
**"AI를 혼내지 말고 BRAIN.md를 고친다"** 가 운영의 핵심 감각입니다. 규칙은 `BRAIN.md` 한 곳만 고치면 세 AI에 동시에 적용됩니다.

### 9.4 명령어 추가·수정
`.claude/commands/` 폴더에 `.md` 파일을 만들면 그게 곧 새 명령어가 됩니다. 파일 이름이 명령어 이름입니다.
자주 쓰는 긴 부탁이 있으면 여기에 넣어 `/이름`으로 줄이세요.

### 9.5 남에게 넘겨줄 때 (빈 템플릿 만들기)
내 개인정보를 지우고 남 주려면 세 곳을 비우면 됩니다:
- `BRAIN.md`의 프로필 → 빈칸으로
- `brain/index.md`의 내 PC 경로 → 일반 경로로
- `brain/lessons.md`의 개인적 교훈 → 삭제

## 10. 자기개선 루프 (lessons)

`brain/lessons.md`는 세 AI가 공유하는 **교훈 노트**입니다.
사용자에게 교정받거나 같은 실수가 반복되면 여기에 한 줄 남깁니다. 모든 AI는 큰 작업 전에 이걸 읽습니다.
같은 교훈이 3번 이상 쌓이면 아예 `BRAIN.md` 규칙으로 승격합니다.
**혼내는 대신 기록하는 것** — 이게 시스템이 시간이 지날수록 똑똑해지는 이유입니다.

## 11. 안전·백업

1. **승인 습관**: AI가 파일 삭제·이동을 물으면 반사적으로 Yes 하지 않습니다. 특히 삭제·대량 이동은 내용 확인 후 승인.
2. **민감정보 금지**: 비밀번호·주민번호·금융정보·토큰은 뇌에 넣지 않습니다. **AI가 읽는 폴더 = 외부로 전송될 수 있는 폴더**라고 간주하세요.
3. **백업**: 이 폴더는 git 저장소입니다. 주간정리 때 커밋해 두면, AI가 뭔가 잘못 정리해도 이전 시점으로 되돌릴 수 있습니다. (클라우드 동기화 폴더에 두면 이중 안전.)
4. **자동화 제외**: 삭제·대량 이동은 절대 자동화하지 않습니다.

## 12. 자주 묻는 질문 / 문제 해결

**Q. 옵시디언에서 파일이 안 보여요.**
`HQ` 전체가 아니라 **`brain` 폴더**를 볼트로 열었는지 확인하세요.

**Q. `/ingest` 같은 명령어가 안 떠요.**
클로드코드를 **`HQ` 폴더**에서 열었는지 확인하세요. 다른 폴더에서 열면 명령어가 안 보입니다.

**Q. AI가 규칙을 무시해요.**
`BRAIN.md`에 그 규칙이 있는지 확인하고, 없으면 추가하거나 문장을 더 구체적으로 고칩니다.

**Q. 안티그래비티(제미나이)가 규칙을 모르는 것 같아요.**
"BRAIN.md 먼저 읽어줘"라고 한 번 말하면 됩니다. GEMINI.md에 그 안내가 있어 대부분 자동으로 됩니다.

**Q. 폴더가 너무 많아 헷갈려요.**
전부 안 써도 됩니다. 필요한 것만 쓰고 나머지는 비워두세요. 나중에 AI에게 "안 쓰는 폴더 정리해줘" 하면 됩니다.

**Q. 용량이 걱정돼요.**
글은 순수 텍스트라 노트 1만 개가 사진 10장 수준(~30MB)입니다. 용량을 먹는 건 첨부 이미지·PDF뿐이니, 큰 원본은 뇌 밖에 두고 요약+링크만 남기세요.

## 14. 네 번째 문 — ChatGPT(코덱스) 연결하기

지금 HQ는 옵시디언·클로드코드·안티그래비티(제미나이) 세 문만 연결돼 있습니다. ChatGPT(코덱스)는 4장에서 본 것처럼 **검증 담당(제3의 시선)**으로 나중에 붙일 예정인 네 번째 문입니다. 이 장은 그 문을 실제로 여는 날, 바로 따라 할 수 있도록 미리 적어 둔 절차입니다.

### 14.1 왜 코덱스인가

코덱스(Codex CLI)는 OpenAI가 만든 **터미널 AI 에이전트**입니다. 터미널(글자로 명령을 입력하는 검은 창)에서 실행한다는 점에서, 지금 HQ의 제작 메인으로 쓰는 클로드코드의 "GPT판"이라고 보면 됩니다. 클로드코드가 프로젝트 폴더의 `CLAUDE.md`를 읽어 규칙을 따르듯, 코덱스는 `AGENTS.md`라는 표준 규칙 파일을 자동으로 읽습니다. HQ 루트에는 이미 `AGENTS.md`가 준비돼 있고("먼저 BRAIN.md를 읽어라", "검증 담당, 읽기 전용, 결과는 `_리뷰.md`로 저장, 고쳐쓰기 금지"), 그래서 **코덱스는 설치만 하면 바로 HQ의 규칙 안에서 움직일 수 있는 상태**입니다.

### 14.2 연결 절차 개요

사람이 직접 해야 하는 단계와 AI가 대신할 수 있는 단계를 나누면 이렇습니다.

1. **구독/계정 확인.** 코덱스는 ChatGPT 계정만 있으면 요금제와 무관하게 쓸 수 있고(무료 요금제 포함, 다만 사용량 한도가 낮음), 구독 없이 API 키로 쓰는 방법도 있습니다. 어떤 방식이 내 상황에 맞는지는 실행 시점에 확인이 필요합니다.
2. **Codex CLI 설치.** 공식 설치 스크립트를 내려받아 실행합니다(정확한 명령은 실행 시점의 공식 문서 기준으로 재확인).
3. **로그인.** ChatGPT 계정으로 로그인하거나 API 키를 등록합니다.
4. **HQ 폴더에서 열기.** 터미널에서 내 HQ 폴더(예: `C:\Users\<사용자명>\HQ`)로 이동한 뒤 `codex` 명령으로 실행합니다.
5. **`AGENTS.md`가 자동으로 읽히는지 확인.** 코덱스에게 "지금 어떤 규칙을 따르고 있어?"라고 물어 `AGENTS.md`와 `BRAIN.md` 내용을 인지하고 있는지 확인합니다.
6. **검증 워크플로우 리허설.** 뇌 안의 샘플 파일 하나를 골라 "검토해줘"라고 시켜 보고, 같은 폴더에 `원본파일명_리뷰.md`가 새로 생기는지, 원본은 그대로인지 확인합니다.

### 14.3 붙여넣기용 프롬프트 — 클로드코드(오퍼스)에게 그대로 맡기기

아래 지시문을 나중에 클로드코드(오퍼스로 상승 실행 권장)에 그대로 복사해 붙여넣으면, 위 6단계를 오퍼스가 대신 진행하면서 사람이 직접 해야 하는 부분(설치·로그인)만 단계마다 확인받습니다.

```
너는 <내 HQ 폴더 경로>에서 작업한다. 먼저 BRAIN.md와 AGENTS.md를 읽고 공용 규칙과
코덱스 어댑터 내용을 파악해라.

이 작업의 목표는 ChatGPT(코덱스)를 HQ의 네 번째 문(검증 담당)으로 실제 연결하는 것이다.

1. Codex CLI의 오늘 날짜 기준 최신 설치 방법, 요금제/구독 조건(무료 요금제로
   되는지, ChatGPT Plus 이상이 필요한지, API 키 방식과의 차이), AGENTS.md 지원
   방식(자동으로 읽는지, 파일 이름·위치 규칙이 바뀌었는지), 커스텀 프롬프트
   (슬래시 명령어) 등록 방법과 저장 위치를 웹 검색으로 먼저 확인해라. 훈련
   데이터 기억으로 답하지 말고, 반드시 오늘 시점 검색 결과로 확인해라.

2. 설치와 로그인은 사용자가 직접 해야 하는 단계다. 실행할 명령어와 이유를
   안내하고, 각 단계마다 사용자에게 진행 여부를 확인받은 뒤에만 다음 단계로
   넘어가라. 절대 사용자 대신 설치나 로그인을 진행하려 하지 마라(할 수도 없다).

3. 연결이 끝나면 HQ 폴더에서 코덱스를 열어 다음을 테스트해라.
   - BRAIN.md와 AGENTS.md의 규칙(검증 담당, 읽기 전용, 결과는 `_리뷰.md`로
     저장, 고쳐쓰기 금지)을 코덱스가 인지하고 있는지.
   - 뇌 안의 샘플 파일 하나를 골라 코덱스에게 검토를 시키고, 같은 폴더에
     `원본파일명_리뷰.md`가 새로 생기는지, 원본 파일은 수정되지 않았는지.

4. 클로드 전용 명령어 7종(ingest/query/lint/save/아침브리핑/저녁마감/주간정리)
   중 코덱스에게도 필요한 게 있는지 판단해라. 코덱스는 검증 담당이라 보통은
   필요 없을 것이다. 정말 필요한 게 있으면 코덱스의 커스텀 프롬프트 방식으로
   옮겨 심어라.

5. 실제로 확인한 결과(설치 방법·요금제·AGENTS.md 동작 방식 등)가 사용설명서
   14장의 서술과 다르면, 14장을 현실에 맞게 갱신해라. 확인하지 못한 부분은
   추측으로 채우지 말고 "확인 필요"로 남겨라.

완료 기준: 코덱스가 HQ 폴더에서 열리고, BRAIN.md 규칙을 인지하며, 샘플 파일
검토 리허설에서 `_리뷰.md`가 정상 생성되고 원본은 그대로 남는다.
```

### 14.4 주의사항

- 코덱스는 **검증 담당**입니다. 쓰기 권한은 최소한으로 유지하고, 뇌나 프로젝트 파일을 직접 고치게 두지 않습니다.
- **담당제**를 지킵니다. 같은 질문을 클로드·제미나이·코덱스 세 곳에 동시에 던지지 않습니다. 조사는 제미나이, 제작은 클로드, 밖으로 나가는 최종본 검증만 코덱스입니다.
- 실제 연결 전에 **구독/사용량 한도**를 먼저 확인하세요. 요금제와 한도는 자주 바뀌므로, 이 장에 적힌 내용보다 연결 시점의 공식 문서를 우선합니다.
